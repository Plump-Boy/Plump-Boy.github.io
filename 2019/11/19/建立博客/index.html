手把手教你搭建个人博客

2019年11月19日 1,199 字 大概 5 分钟 博客 博客相关

之前看了豆哥的博客搭建教学，自己也想搭一个玩玩，奈何个人技术是在太差，看不懂豆哥那个的操作流程，又去网上找教程结果又踩坑了，最终兜兜转转一天的功夫才搞好，现在与大家分享一下流程，也让大家少走弯路。

1.准备工作

1.注册一个GitHub账号

2.下载并安装node.js(https://nodejs.org/en/)

3.下载并安装git(https://git-scm.com/)

4.如果你想拥有一个属于自己的域名的话，提前去阿里云上面进行实名认证并且买一个域名（一般一年三四十块钱）

2.配置SSH key

此步骤可做可不做，GitHub配置SSH Key的目的是为了帮助我们在通过git提交代码是，不需要繁琐的验证过程，简化操作流程。
1.打开git bash（之前下载git后可以找到这个程序），第一次使用要配置一下，输入下列代码

git config --global user.name "xxxxx"
git config --global user.email  "xxxxx@xxxxx.xxx"

2.输入

cd ~/.ssh
ls
ll

检查有没有id_rsa 和 id_rsa.pub文件，如果存在则说明已经有SSH key，如果没有则需要生成一个。

ssh-keygen -t rsa -C "xxxxx@xxxxx.xxx"

输入完后应该会提示一些东西，不用管，敲三下回车。
此时进入C:\Users\你的用户名\.ssh里面，找到id_ras.pub文件并且用记事本打开它，复制里面所有的内容。

这时打开GitHub，点击setting

再点击SSH and GPG keys

再点击New SSH key


最终在key内粘贴之前复制的内容（title随便写），点击Add SSH key即可s。

3.最后进行检查，再git bash中输入

ssh -T git@github.com

运行结果出现类似如下

Hi xxxxx! You've successfully authenticated, but GitHub does not provide shell access.

那么SSH key就已经配置成功了。

3.安装Hexo

1.在你喜欢的地方新建一个文件夹，并且在git bash里面进入这个文件夹（cmd命令请自行百度），输入

npm install -g hexo


接着输入，此处会有些慢，要等待一下。

hexo init


然后输入

hexo s

会启动本地的4000端口，http://localhost:4000 打开这个你就能在本地看到你的博客了。注意在看完之后返回git bash摁ctrl+c关闭。如果没有可以用hexo clean命令清除一下，再hexo g生成一次，再hexo s尝试。当然也有可能是4000端口占用，解除方法百度。

4.把博客推到GitHub库中

1.打开github，点击your repositorise

点击右边的new新建一个库

特别注意！！！库的名称一定要是:你的github用户名.github.io，最后点击create repositorise完成创建。

2.在git bash中输入代码，等待安装插件完成。

cnpm install --save hexo-deployer-git

3.在这个目录下面用记事本打开config.yml这个文件，在最下面deploy那里，全部改成（注意中间的空格！）

deploy；
  type: git
  repo: 你的库的地址
  branch: master

然后保存并退出。
最后在git bash里面输入

hexo d

可能还要输入一次github账号和密码。

4.刷新github的库的页面，会发现多了很多东西，在浏览器里面输入刚刚创建的库名称，我们的博客就可以显示出来了，博客搭建成功！

5.绑定域名

1.打开阿里云控制台，在域名管理里面你的域名后面点击解析

2.点击添加记录
创建两次，记录类型一定要选择CNAME!!主机记录一次www，一次@，记录值就是你的个人博客的地址（去掉https：//其实跟仓库名称是一样的）

确定之后在github库的页面点击setting，

在下面的github pages里面的Custom domain写下你的域名，

这样就可以用域名访问你的博客了。至于后面的更换主题写博客之类的就自己去研究了。

写在最后

现在回过头来写确实感觉步骤十分简单，可就是这样的步骤在我搭建的时候错误也是不断，你现在看到的是一个混杂版，前半部分是一个网上的教程，可惜他后半部分是错的，所以当初在搭建的时候就一直没法成功，后来改成了codesheep的方法搭建后半部分才得以成功。对我感触最大的应该是用百度好处了，在搭建过程中出现的一个又一个问题基本上都能在百度上找到答案（虽然也有些没能解决）。在我看来，理解每一步是在干啥，能收获的比结果更多。
